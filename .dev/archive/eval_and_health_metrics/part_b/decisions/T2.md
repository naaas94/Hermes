# T2 — Structured logger with dual sink and config flag

## Chosen approach

- **`hermes/obs/logging.py`**: `configure_logging(config, verbose=...)` clears Hermes-tagged root handlers (`hermes_obs_stream`, `hermes_obs_ndjson`) and re-attaches handlers so repeated calls are idempotent in tests and CLI re-entry. Console uses the legacy stdlib format `%(name)s %(levelname)s: %(message)s` unless `log_format="json"`, in which case structlog’s `ProcessorFormatter` + `ConsoleRenderer(colors=False)` render the stream (NDJSON remains a separate file sink).
- **`log_format="json"`** requires structlog (`[obs]`); **`HermesObsExtraRequired`** is raised at startup if JSON mode is requested without the extra (v0.2 error envelope). **`log_ndjson=True`** with `log_format="console"` appends NDJSON via **`HermesNDJSONHandler`** without requiring structlog.
- **NDJSON path**: `<resolved log_dir>/hermes-<YYYYMMDD>.ndjson`; `log_dir` is relative to `get_storage_base()` when not absolute. Lines are JSON objects with `schema_version` (from **`CURRENT_LOG_SCHEMA_VERSION`**), `ts`, `level`, `logger`, `message`, and optional `job_id` / `trace_id` from context.
- **`bind_job`**: `contextvars` dict for stdlib `LogRecord` injection via **`_HermesObsContextFilter`**; when structlog JSON mode is active, also **`structlog.contextvars.bound_contextvars`** so structlog merges job context.
- **`get_logger`**: Returns structlog’s logger only when JSON mode is active; otherwise **`_StructlogShim`** around **`logging.getLogger`** so `.bind()` exists without structlog.
- **`hermes/cli.py`**: Typer callback calls **`configure_logging(load_config(), verbose=...)`** instead of bare **`basicConfig`**, preserving default console behavior for **`log_format="console"`**.
- **`hermes/config.py`**: **`ObservabilityConfig`** with **`log_format`**, **`log_dir`**, **`log_ndjson`** under **`[observability]`** in `config.toml`.

## Alternatives rejected

- **stdlib-only JSON logging for `log_format=json`** — rejected in v0.2 (T1.1 decision): doubles maintenance and risks drift vs structlog NDJSON; JSON mode is locked to **`[obs]`**.
- **structlog console renderer for default `log_format=console`** — rejected to preserve existing CLI logging appearance; structlog console integration is limited to **`log_format=json`**.

## Assumptions (load-bearing)

- Root logger may be reconfigured by **`configure_logging`**; third-party libraries that mutate root handlers outside Hermes tags are unaffected by our clear step.
- NDJSON write failures are rare; one stderr warning and continued console logging match the error envelope.

## Deferred

- **Perf regression measurement** (dual sink vs old `basicConfig` on smoke workloads): not gated in CI; if >10% regression appears, consider an async NDJSON queue or sampling (kill criterion 3 in packet).
- **README rotation/cleanup** for NDJSON files: documented at plan level for Part B docs tasks; no in-code rotation in v1.
