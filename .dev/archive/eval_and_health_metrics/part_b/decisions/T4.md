# T4 — `hermes bench` harness (workloads, summary, CSV/JSON)

## Chosen approach

- **`hermes/bench/runner.py`**: Pydantic **`BenchWorkload`**, **`BenchResult`**, **`BenchSummary`**; **`run_bench`** runs the real **`run_pipeline`** path with optional **`--mock-llm`** (same **`MagicMock`** pattern as **`tests/test_pipeline_integration.py`**). Metrics come from wall time, DB aggregates (tokens, validation, page/chunk counts), **`rss.sample`** peaks collected in-process via a small **`logging.Handler`** that parses JSON **`event`** lines from the obs stack, plus **`resource.getrusage`** when the **`resource`** module exists (Unix).
- **Dual sink comparison**: When **`structlog`** is available (`[obs]`) and compare is enabled (per-workload **`compare_log_format`** for **`pdf_text_small`**, or **`--log-format-compare/--no-log-format-compare`**), each workload runs twice with **`configure_logging`** toggling **`observability.log_format`** **`console`** vs **`json`**. **`dual_sink_overhead_pct`** is wall-time overhead of the JSON leg vs console; **`pdf_text_small`** triggers **`bench.dualsink.regression`** at WARN and sets **`BenchSummary.dual_sink_regression`** when overhead **> 10%** (CLI exits non-zero).
- **`hermes/bench/workloads.py`**: **`default_workloads`** resolves **`tests/fixtures/eval`** first, then generated top-level **`tests/fixtures/`** paths; optional **`test_pdf_stress_riscbac.pdf`** at repo root with **`--include-large`**.
- **`bench.*` events**: Emitted as JSON strings through **`get_logger("hermes.bench.harness")`** using schema models (**`bench.workload.start`**, **`bench.workload.end`**, **`bench.summary`**). **`bench.dualsink.regression`** is a **warning log line** (not a new **`EventName`**) to avoid a schema bump in T4’s file allow-list.
- **CLI**: Typer **`bench`** subcommand alongside existing commands; **`configure_logging`** is reapplied after **`run_bench`** because the runner calls **`reset_logging_for_tests`**.

## Alternatives rejected

- **Tailing NDJSON on disk for peak RSS** — rejected in favor of an in-process log collector plus optional **`getrusage`** cross-check (packet preference: subscriber over tail).
- **Adding `bench.dualsink.regression` to `EventName`** — rejected to stay within T4’s declared **files to touch** (schema unchanged); WARN string carries the acceptance signal for tests and operators.

## Assumptions (load-bearing)

- **`run_pipeline`** remains the single execution path; bench does not fork a lighter “fake” pipeline for real (non-mock) runs.
- Dual JSON runs require **`structlog`**; without **`[obs]`**, compare is skipped with a one-time warning and a single console attempt per workload.
