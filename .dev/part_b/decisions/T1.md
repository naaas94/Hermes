# T1 — Log schema & event catalog (decision log)

## Chosen approach

- **`hermes/obs/schema.py`**: PEP 695 `type` aliases for `LogSchemaVersion`, `EventName`, and `StageName`; `CURRENT_LOG_SCHEMA_VERSION` set to `"1.0"`.
- **`BaseEvent`** and one Pydantic model per `event` value, combined into a **discriminated union** (`HermesLogEvent`) on `event` for strict validation without an oversized single `Literal` union field list.
- **`EVENT_FIELD_CATALOG`**: documents required / optional payload keys (beyond common `BaseEvent` fields) and supports tests that every `EventName` is catalogued.
- **`validate_event(dict) -> bool`**: guards `schema_version` shape (`major.minor`), membership in known `event` strings, catalog required keys, then `TypeAdapter(HermesLogEvent).validate_python`; never raises.
- **PEP 695 + `get_args`**: runtime lists use `_literal_union_args()` (via `__value__`) so `typing.get_args` works on `type EventName = Literal[...]`.
- **`pyproject.toml`**: optional extra `[obs]` with `structlog` and `psutil` per packet (pins confirmed as minimum versions).

## Alternatives rejected

- **Single giant `Literal` on one `BaseEvent` field** — rejected to avoid unwieldy unions and to keep per-event models testable; event count is small either way.
- **TypedDict-only catalog** — rejected as primary because the packet calls for Pydantic `BaseEvent` and discriminated validation; TypedDict would duplicate shape work for T2–T4 consumers.
- **Aligning `StageName` strings with SQLite `pipeline_stages.stage` values** (`normalization`, `chunking`, `extraction`) — rejected in favor of the locked Part B contract (`normalize`, `chunk`, `extract`, …); emitters can map DB strings to logging names.

## Assumptions (load-bearing)

- Downstream tasks import **`EventName`**, **`StageName`**, **`BaseEvent`**, **`HermesLogEvent`**, and **`CURRENT_LOG_SCHEMA_VERSION`** from `hermes.obs.schema` (or `hermes.obs`) and do not redefine event strings inline.
- **`schema_version`** in NDJSON remains semver-ish `major.minor` until a future major revision.

## Deferred

- Wiring loggers, NDJSON sinks, RSS sampling, and bench commands (T2+).
- **`trace_id`** propagation beyond optional field presence on `BaseEvent`.
