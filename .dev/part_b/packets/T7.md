## Task statement + non-goals

Hermes markets itself as **memory-safe** (bounded RAM, streaming, page-at-a-time) and **scalable** (workload size, parallelism with cloud, WAL SQLite). Part B turns those claims into measurable, reproducible artifacts by layering observability + a benchmark harness on top of the existing pipeline:

1. A **fixed, versioned log schema** covering stage boundaries, RSS samples, throughput counters, and LLM cost proxies.
2. **Structured logging** (`structlog`) with a **dual sink**: human-readable console (existing UX preserved) + NDJSON file under a configurable logs directory, behind a config flag `log_format = "json" | "console"`.
3. An **RSS sampling helper** wired at pipeline stage transitions (preflight / normalization / chunking / extraction) with **optional `psutil`** and graceful degradation when absent.
4. A **benchmark command** (`hermes bench` or documented script) that executes a standard workload set and emits a summary of peak RSS, wall time, throughput (pages/rows/chunks per minute), token totals, and cost proxy.
5. **Docs** — a README "Benchmarks & memory" subsection with a reference-hardware table and methodology so claims are checkable.
6. **CI artifact** — upload the bench summary JSON on main-branch runs (best-effort; flaky regression gates explicitly out of scope).

**Non-goals (unchanged in v0.4, plus extra bindings for T7):**

- **Regression gates / CI fail-on-threshold** — still out of scope.
- **Full OpenTelemetry** — still out of scope.
- **Remote metric sinks** — local NDJSON is the v1 path.
- **Chart generation in-process** — charts generated manually per release.
- **Time-series storage of RSS inside SQLite** — NDJSON is the artifact.
- **Changes to extraction, normalization, or LLM-call semantics** — T7 is observability-adjacent config plumbing + docs only.
- **Schema bump** — T7 adds fields to `ObservabilityConfig` (a `hermes/config.py` dataclass), *not* to any NDJSON event shape. `LogSchemaVersion` stays at `"2.0"`.
- **Structural README rewrite** — T7 appends a short paragraph to T5's "Benchmarks & memory" section; it must not reshape the section's heading, sibling sections, or the reference-hardware table.
- **Resurrecting the stdlib-JSON fallback** — the `HermesObsExtraRequired` contract from v0.2 stands. T7 does not reintroduce a fallback path for missing `[obs]`.
- **Fixing audit F3 / F4 / F7** — those are executor-owned decision-log / packet-prose hygiene items tracked separately in the v0.4 changelog. T7 does not touch `decisions/T1.md`, `decisions/T4.md`, or the `normalize`/`chunk`/`extract` prose residue in frozen packets.

### Error envelope

| Case | Behavior |
|------|----------|
| `psutil` not installed | `sample_rss()` emits one `warn` event (`rss.sample.unavailable`) on first call, then no-ops. |
| `structlog` not installed | **No stdlib-JSON fallback.** If `log_format="json"` without the `[obs]` extra, `configure_logging()` raises `HermesObsExtraRequired` at startup with an install hint. Console sink remains available via stdlib without structlog. |
| NDJSON sink write fails | Log once to stderr; continue console-only; never block pipeline. |
| Invalid `log_format` value | Fall back to `"console"`; log a config warning. |
| `hermes bench` workload fails mid-run | Emit `bench.workload.end` with `status="error"`; continue other workloads; non-zero exit at end. |
| Missing fixture for bench | Skip with `status="skipped"`. |
| `BenchResult` missing optional fields | Serialize as `null`. |

### Logging

- **Dual sink mandatory:** console always on; NDJSON on when `log_format="json"` (or via `log_ndjson=True`).
- **Correlation:** every event carries `job_id`; bench events also carry `bench_run_id`.
- **Schema versioning:** `schema_version` on every event; major bump on breaking changes.
- **Sampling cadence:** RSS sampled at stage boundaries (`preflight`, `normalization`, `chunking`, `extraction`) and optionally on a timer during long normalization (`rss_sampling_interval_s > 0`). **T7 is the subtask that makes the timer setting actually configurable from user TOML.**
- **No secrets in events.**
- **Logger names:** `hermes.obs.*`, `hermes.bench.*`.
- **Level policy:** structured events at `INFO`; failures at `ERROR`; degradation warnings at `WARN`, once per process.

### Tests

- Framework: **pytest** (existing).
- Location: `tests/test_obs_*.py`, `tests/test_bench_*.py`.
- Naming: `test_<module>_<behavior>`.
- T7 coverage expectation: at least one TOML-round-trip test asserting that non-default `rss_sampling_enabled` / `rss_sampling_interval_s` values propagate from user TOML to `ObservabilityConfig`. Existing `ObservabilityConfig` tests that assert a closed field set (if any) must be updated to the new five-field set.
- Determinism: no RSS or timing assertions in T7's tests (config plumbing is deterministic; timing is T4's domain).
