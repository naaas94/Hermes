# Changelog

## 2026_04_10

- **PEP 561** (`hermes/py.typed`): empty marker so the package is treated as typed when installed. **Rationale:** without it, tools like mypy may not use `hermes`’s annotations in dependent projects (PEP 561).
- **`hermes init`** now installs editable example schema modules under `~/.hermes/hermes_user/examples/` (`vehicle_fleet.py`, `generic_table.py`), copied from the bundled examples only when each file is missing, so user edits are preserved on re-run.
- **New config from init** sets `default_schema` to `hermes_user.examples.generic_table:GenericRow` instead of the site-packages path, so extraction works from the user directory without touching installed packages.
- **Schema loading** prepends `~/.hermes` to `sys.path` before importing modules, so `hermes_user.examples.*` resolves without setting `PYTHONPATH`.
- **Documentation** (`README.md`) updates the Quick Start with the user-local schema paths and clarifies custom schema layout versus `~/.hermes`.
- **Tests** add coverage for user schema installation and loading (`tests/test_user_schemas.py`).
- **`hermes extract --pages`** (`hermes/cli.py`): optional page/sheet subset for large documents; help text documents 1-based PDF page numbers and Excel **sheet** indices (not rows); wired via `pages_spec` into `run_pipeline`.
- **Page spec parsing** (`hermes/ingestion/pages_spec.py`): supports ranges and lists (e.g. `1-10`, `3,5,7`); `resolve_page_indices_0` validates against preflight totals and yields 0-based indices for normalizers.
- **Pipeline** (`hermes/extraction/pipeline.py`): resolves `--pages` after preflight; passes `page_indices` to the router; preflight/normalization stage details note subset size; fails clearly if normalization returns no pages; chunking uses `normalized_pages`; `jobs.page_count` remains full document size.
- **Normalization** (`hermes/normalization/router.py`, `pdf_text.py`, `pdf_ocr.py`, `excel.py`): optional `page_indices` filters PDF iteration and Excel sheets; output still uses stable `page_{n}.md` / `sheet_{n}.md` names and original 0-based indices for `NormalizedPage` / chunk `source_pages` (DLQ chunk paths unchanged).
- **Normalization progress** (`hermes/extraction/pipeline.py`, router + normalizers): Rich progress bar for the normalize stage (same columns as extraction); normalizers accept optional `on_page_done(page_idx)` and invoke it after each sheet/page so the pipeline can advance the bar without coupling normalizers to Rich. **Rationale:** OCR/normalization often dominates wall time versus LLM extraction, which already had a bar; users need visible progress for long scans.
- **OCR model caching** (`hermes/normalization/pdf_ocr.py`): Surya detection/recognition models and the EasyOCR reader load once per process via `@lru_cache(maxsize=1)` helpers (`_get_surya_models`, `_get_easyocr_reader`) instead of on every page. **Rationale:** per-page reloads were a major performance bug on multi-page scans; keeping models resident for the process matches typical CLI usage (exit after the run).
- **Tests** (`tests/test_pages_spec.py`, `tests/test_normalization.py`): parser/validation coverage and PDF single-page filter smoke test.
- **Chunk token estimates** (`hermes/normalization/chunker.py`): optional `tiktoken` for tokenizer-accurate `estimate_tokens` and token-window splits when installed; falls back to the `CHARS_PER_TOKEN` heuristic if `tiktoken` is absent or `get_encoding` fails (e.g. unknown encoding name).
- **Token heuristic alignment** (`hermes/normalization/chunker.py`): `CHARS_PER_TOKEN` is **4**, matching preflight’s `total_chars // 4` (`hermes/ingestion/preflight.py`) and common English/Spanish character-per-token guidance. **Rationale:** one consistent estimate for chunk merge/split and cost preflight; the previous value (2) skewed chunking toward more, smaller segments than preflight implied. Hard caps (`MAX_TEXT_CHUNK_TOKENS`, `MAX_TABLE_CHUNK_TOKENS`) unchanged.
- **Config** (`hermes/config.py`): `extraction.tiktoken_encoding` (default `cl100k_base`) selects the tiktoken encoding when the optional dependency is present.
- **Packaging** (`pyproject.toml`): optional extra `tiktoken` (`tiktoken>=0.7`).
- **Documentation** (`README.md`): install note for the `tiktoken` extra and `extraction.tiktoken_encoding`.
- **Dependencies** (`pyproject.toml`): removed unused `tenacity`; the extraction validator never imported it. **Rationale:** `hermes/extraction/validator.py` already implements repair retries with a plain `while` loop and `max_retries` (from `cfg.llm.max_retries` via `hermes/extraction/pipeline.py`)—bounded attempts, no inter-attempt backoff. Keeping an extra dependency would not change behavior unless we added waits/backoff, which would risk rate limits and test timing drift.
- **Documentation** (`.dev/Hermes.md`): tech stack and memory-safety tables now describe that bounded manual repair loop (default `max_retries=2` → up to three LLM attempts per chunk) instead of incorrectly citing `tenacity` and exponential backoff.
- **Python version:** `.dev/Hermes.md` now states Python 3.12+ to match `requires-python = ">=3.12"` in `pyproject.toml`. Ruff (`target-version`) and Mypy (`python_version`) are set to 3.12 for consistency with that minimum.
- **Packaging** (`pyproject.toml`): `[tool.hatch.build.targets.wheel]` sets `packages = ["hermes"]` so the wheel includes SQL migrations under `hermes/migrations/`. **Rationale:** explicit wheel target avoids relying on implicit defaults and documents that migrations ship with the package.
- **Migrations layout:** SQL files moved from repo-root `migrations/` to `hermes/migrations/` so installed and editable layouts match the package tree.
- **Config** (`hermes/config.py`): `get_migrations_dir()` resolves to `Path(__file__).parent / "migrations"` instead of `parent.parent / "migrations"`. **Rationale:** the old path pointed at a non-package location that disappears after `pip install`, so `hermes init` could skip applying migrations silently.
- **Database connections** (`hermes/db.py`): added `open_db()` and `open_connection()` context managers (via `contextlib.contextmanager`) that wrap `init_db()` and `get_connection()` and always call `conn.close()` in `finally`. **Rationale:** matches manual open/close semantics but guarantees cleanup on exceptions and removes scattered `conn.close()` on every early-return path.
- **CLI** (`hermes/cli.py`): `test`, `status`, `retry`, `export`, and `init` use `with open_db()` (or a no-op body where only initialization matters) instead of bare `init_db()` + `close()`.
- **Pipeline** (`hermes/extraction/pipeline.py`): `run_pipeline()` holds the main job connection under `with open_db()`; `_process_chunk()` uses `with open_connection()` per chunk so parallel workers still get one connection each without re-running migrations every time. **Rationale:** same thread-safety as before (`get_connection` per chunk), with reliable teardown.
- **Database** (`hermes/db.py`, `hermes/extraction/pipeline.py`): `save_result`, `save_llm_run`, `save_pipeline_stage`, and `save_failed` accept `commit` (default `True`); `_process_chunk` passes `commit=False` and issues a single `conn.commit()` after all chunk writes. **Rationale:** per-call commits meant hundreds of WAL syncs per large job; one commit per chunk preserves chunk-level durability and matches DLQ retry semantics if a run dies mid-chunk.
- **Pipeline graceful shutdown** (`hermes/extraction/pipeline.py`): SIGINT sets a threading `Event`; the extraction loop checks it before each chunk (sequential) or before submitting further work (threaded). On interrupt, the extraction stage is recorded as interrupted, the job is set to **partial** if any chunk succeeded else **failed**, the DB connection closes via the existing `open_db` context, and a short progress summary is printed; threaded runs use `ThreadPoolExecutor.shutdown(wait=False, cancel_futures=True)`. **Rationale:** Ctrl+C should not leave the job stuck in `extracting` or require killing the process; incremental submit + cancel avoids queuing all chunk futures at once so shutdown can skip not-yet-started work.
- **Pipeline** (`hermes/extraction/pipeline.py`): added `_fail_job()` for early failure paths (unsupported file type, normalization exception, empty pages after normalization)—single place for saving the pipeline stage, setting `JobStatus.FAILED` with `normalization_error`, and printing the red error line. **Rationale:** removes three copies of the same sequence; behavior unchanged.
- **CI** (`.github/workflows/ci.yml`): GitHub Actions workflow on push to `main` and `dev` and on all pull requests—Python 3.12, editable install with `[dev]`, then `ruff check .`, `mypy hermes/`, `python tests/generate_fixtures.py`, and `pytest`. **Rationale:** fixtures are gitignored, so CI must generate them before tests; OCR-heavy tests remain skipped without optional `ocr` extras, matching the intended P2 scope; running on `dev` catches failures on the integration branch (including direct pushes) before merging to `main`.
- **Dev typing** (`pyproject.toml`): `types-openpyxl` added to `[dev]` for strict mypy against `openpyxl`; mypy overrides `ignore_missing_imports` for `surya.*`, `easyocr`, and `PIL.*` because those ship only with the optional `ocr` extra, not in default/CI dev installs.
- **Typing** (`hermes/config.py`): `import tomllib` only; removed the `tomli` fallback. **Rationale:** `requires-python >= 3.12` guarantees stdlib `tomllib`; the unused fallback imported a non-dependency and made mypy fail with `import-not-found` in CI (`# type: ignore[no-redef]` did not cover that error code).
- **Strict mypy pass** (`hermes/`): annotations and small API tweaks so `mypy hermes/` succeeds under `strict = true`—PyMuPDF calls annotated or narrowly suppressed (`open` is untyped in stubs), JSON parse paths in `validator.py` use `cast`, pipeline/CLI use `setattr` for dynamic LLM `model` and narrow `fail.id` before DLQ updates, `open_db`/`open_connection` generators typed, OCR helpers fully typed, `pdf_ocr` page iteration uses explicit `list[int]`. **Rationale:** keep CI’s type-check step meaningful without installing heavy optional OCR stacks on every run.
- **Lint** (`generate_test_datasets.py`, `hermes/cli.py`, `tests/`): ruff E501 addressed (file-level noqa for the long synthetic-data script; wrapped lines elsewhere) so `ruff check .` matches CI.
- **CLI** (`hermes/cli.py`): global `--verbose` / `-v` (Typer `@app.callback`) configures the root logger via `logging.basicConfig`—DEBUG when verbose, WARNING otherwise (`%(name)s %(levelname)s: %(message)s`). **Rationale:** pipeline, validator, LLM client, and PDF OCR use module loggers at INFO/DEBUG; with no `basicConfig`, everything below WARNING was discarded. Place the flag before the subcommand (e.g. `hermes -v extract …`).
- **Dead code** (`hermes/ingestion/storage.py`, `hermes/schemas/loader.py`): removed unused `read_raw()` and `discover_schemas()`—no callers in the tree. **Rationale:** shrinks the public-ish surface of those modules and avoids carrying dead helpers that suggest supported behavior.
- **Database** (`hermes/db.py`): `delete_job(conn, job_id)` removes rows from `extraction_results`, `llm_runs`, `failed_extractions`, and `pipeline_stages`, then deletes the `jobs` row (child tables first for FK order).
- **CLI** (`hermes/cli.py`): **`hermes clean`** removes a job’s storage directory (`{storage_base}/{job_id}/`) and calls `delete_job` for that id, or **`hermes clean --all`** for every job. **`typer.confirm()`** before destructive work; **`--force` / `-f`** skips confirmation for scripts. **Rationale:** gives a single, explicit way to reclaim disk and DB state without leaving orphan rows or files.
- **Export** (`hermes/db.py`, `hermes/cli.py`): `export_results_as_records` streams `record_json` from SQLite and yields parsed records (generator); `hermes export` writes JSONL or CSV row-by-row instead of building the full file in memory. CSV column order comes from the first record. **Rationale:** scales to large jobs without proportional RAM use for export.
- **CLI** (`hermes/cli.py`): **`hermes retry`** after the replay loop re-queries each affected job; promotion to **completed** (with `completed_chunks = total_chunks`, `failed_chunks = 0`) requires **no pending DLQ rows**, job status **partial** or **failed**, and **`COUNT(DISTINCT chunk_index)` in `extraction_results` equal to `jobs.total_chunks`** (via new `count_distinct_extraction_chunk_indices` in `hermes/db.py`). If the DLQ is empty but fewer chunks have results than `total_chunks` (e.g. graceful SIGINT left chunks never submitted—no DLQ row and no result), status is left unchanged and a short yellow message explains. **Rationale:** an empty DLQ does not imply every chunk was extracted; the previous check could mark the job complete and overstate progress.
- **LiteLLM client** (`hermes/extraction/llm_client.py`): `litellm.completion()` uses `num_retries=3` and `retry_strategy="exponential_backoff_retry"` so retriable failures (including 429) get spaced retries via LiteLLM’s `completion_with_retries` path. **Rationale:** `retry_after` is a `litellm.Router` setting and is not honored by `completion()` (it was effectively a no-op); `retry_strategy` is the supported knob for backoff on the completion API.
- **Config** (`hermes/config.py`): `_parse_config` uses `.get("litellm", {})` instead of `.pop()` and skips `litellm` when building `llm_fields`, so the loaded dict is not mutated and the raw nested table is not passed into `LLMConfig` (only the parsed `LiteLLMConfig` is). **Rationale:** callers/tests may reuse the same structure; mutation was easy to miss because `load_config` caches the result.
- **Typing** (`hermes/config.py`): `_parse_config(raw: dict[str, Any])` replaces a bare `dict` plus `# type: ignore[type-arg]`. **Rationale:** strict mypy expects explicit generic parameters on `dict`.
- **Typing** (`hermes/normalization/excel.py`): `_write_sheet_markdown` annotates the worksheet as `openpyxl.worksheet.worksheet.Worksheet` under `TYPE_CHECKING` so openpyxl is not imported at module import time. **Rationale:** removes `no-untyped-def` suppression while keeping the lazy `import openpyxl` inside `normalize_excel`.
- **Typing** (`hermes/normalization/pdf_ocr.py`): `_PdfPageLike` / `_PdfDocLike` / `_PixmapLike` `Protocol`s describe the PyMuPDF surface this module uses; `_ocr_page` and `_render_and_ocr` take `_PdfPageLike` instead of untyped parameters. **Rationale:** PyMuPDF exposes `Page` as a string forward declaration, so `pymupdf.Page` is not a valid mypy type; structural typing matches real usage. `pymupdf.open` keeps a narrow `# type: ignore[no-untyped-call]` because stubs still leave that entry point untyped under strict mypy.
- **Lint** (`hermes/models.py`): `FileType`, `JobStatus`, and `DLQStatus` now inherit from `enum.StrEnum` instead of `(str, Enum)`. **Rationale:** satisfies Ruff UP042 and matches Python 3.11+ guidance for string-valued enums.
- **Lint** (`hermes/extraction/pipeline.py`): wrapped long `_fail_job(...)` calls to stay within the 100-character line limit (Ruff E501).
